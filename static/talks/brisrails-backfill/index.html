>how do we ship?
⏻~67 commits to master in a working day.
⏻Prod is scrubbed, then cloned into staging & development.
⏻We use structure.sql, not schema.rb.
⏻One shared dev database — across every developer.

>the backfill is killing prod
⚠I queued one backfill across every org. It fanned out into hundreds of jobs.
⚠Minutes later: CPU pinned, replication lag climbing, queries timing out.

>diff BackfillRecords{ActiveJob,KeysetJob}
OLD · ActiveJob
NEW · Keyset
pagination
offset — page + per
keyset — WHERE id > last_id
per-batch cost
grows with every page
constant — rides the PK index
kill switch
none — runs to completion
Flipper flag, per job
handle_records
Array (materialized)
AR::Relation → update_all
walk direction
forward only
forward or reverse

OLD · batch(page)
LIMIT 200
OFFSET 19,999,800
 
plan:    Seq Scan + Limit
scanned: 20,000,000 rows
time:    worse every batch
NEW · batch(last_id)
WHERE id > 19,999,800
LIMIT 200
 
plan:    Index Scan
scanned: 200 rows
time:    constant

>the backfill is killing prod, what do I do?
OLD
1. page the on-call
2. find the job in DelayedJob
3. delete each queued run by id
4. hope no orgs were mid-batch
NEW
Flipper.enable(
  :my_backfill_job
)
# that's it. it pauses.
